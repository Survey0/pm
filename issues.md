# US-01
## User Story
**As a** пользователь (неавторизованный посетитель)  
**I want** зарегистрироваться, указав email и пароль  
**So that** получить доступ к созданию и управлению опросами

## Acceptance Criteria
- [ ] На странице /register есть поля: Email, Пароль, Подтверждение пароля
- [ ] Кнопка "Зарегистрироваться" активна только при заполненных полях
- [ ] Валидация email: если введён некорректный email — ошибка "Введите корректный email"
- [ ] Валидация пароля: если пароль короче 6 символов — ошибка
- [ ] Валидация подтверждения: если пароль и подтверждение не совпадают — ошибка
- [ ] При успешной регистрации: пользователь автоматически входит в систему
- [ ] Редирект на /dashboard после регистрации
- [ ] В БД пароль хранится в хешированном виде (bcrypt)
- [ ] При попытке зарегистрировать существующий email — ошибка "Пользователь уже существует"

## Estimate
8 hours

## Priority
Must

## Dependencies
None

# US-13
## User Story
**As a** автор опроса  
**I want** сохранить опрос как черновик, а затем опубликовать его, чтобы получить ссылку для сбора ответов  
**So that** готовить опрос заранее и публиковать только когда он полностью готов

## Acceptance Criteria
- [ ] В конструкторе есть две кнопки: "Сохранить как черновик" и "Опубликовать"

  - Проверка: в опросе есть хотя бы один вопрос

  - Генерируется уникальный публичный slug (например, случайная строка abc123)
  - Пользователю показывается сообщение со ссылкой: "Опрос опубликован! Ссылка: /s/abc123"
- [ ] Опубликованный опрос нельзя редактировать (или можно, но тогда он снова становится черновиком — выберите подход)
- [ ] Ссылка на опрос доступна автору на странице списка опросов и в конструкторе после публикации

## Priority
Must
