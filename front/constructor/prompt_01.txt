We are building a survey creation tool (constructor). User can create surveys which then are saved to the database. For now, don't bother with database, we are doing design phase.

1.png - This is start screen. Actually, screen will be the same, but the page will "grow". "Create new survey" is the button; when clicked, the page shows the following elements (not instead of the start elements, but appended to it).
2.png - Here survey creation begins. We enter its name (into "Survey title" box, which contains prompt "Enter survey title". The field is mandatory. The next field - "Desctiption" - is optional, its prompt is "Enter survey description". The block of buttons underneath can add a question to survey with answers that can be one of: single choice (like radio buttons), multiple choice (like checkboxes) or text block. Buttons Cancel and "Save Survey" shown here are kept at the bottom at all times.
3.png - An example of complete survey before saving. We can see the whole screen from top to bottom. There's "Question 1" with single choice answers (number of answers is not limited, new answer is added to the question on "+ Add option" button. There's check box "Required" to make this question not skippable during survey run. At the top there's static text "Single Choice".
There's "Question 2" with multiple choice answers. The only difference with the previous question palette is the static text saying it's multiple choice. There's no difference in page behavior for those two types of answers (at the design time).
There's "Question 3" with single text block. There's no "+ Add option here" as single text block should be the only way to answer the question.
Each question block has a "delete" button implemented as a waste basket pictogram in the right top corner of each question block.
There can be as many questions in the survey as user wants. Buttons "+ Single Choice", "+ Multiple Choice" and "+ Text" append corresponding question block to the survey between the last question and cancel/Save buttons.
4.png - Shown if we push "Save button". HTML makes an API call (details later) and shows a block which contains info about all existing surveys in the database:
- Survey name
- Survey description
- Number of questions
- Date of creation

Upper right corner contains buttons shown as pictograms:
- "Copy survey link" which populates the clipboard with link (also returned from API call)
- "View results" link which we don't need to implement now
- "Delete survey" with the same waste basket pictogram as before.
This screen is shown instead of all the previous, not appended to previous elements.

If we pressed "Cancel" button instead, all is cleared and we return to the first stage.

That's it.

As for CSS frameworks - I don't care, choose yourself please.
It is desktop design.
As for colors, make them different for the background, buttons and information blocks (survey name+description, questions, final results after save).
Validation: required fields - survey name, question, answers

	"numberOfQuestions": 3,
	"createdAt": "2026-04-06"

The reply is an array. It is because the page actually makes API (REST GET) call to get all currently present surveys from the database. If there are no surveys, then the result is like in 1.png file. If there are surveys, they all are shown like in 4.png file.
