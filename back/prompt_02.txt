Now let's attach some PHP backend stuff.

First, here I upload current PHPs just to show desired approach to writing code.

At the root of backend folder there's bootstrap.php which sets up the environment and loads code files.
Then there's "handlers" directory which contains GetDataHandler.php which is just a sample for all other handler files. All database work should be handled with those handlers (get surveys, delete survey, save survey) in the similar way.
Then there's "config" directory with Database.php file in it. I don't suppose we need to change it in any way; I provided it here just for information (it is referenced from handlers).

Finally, there's get_data.php file which complements GetDataHandler.php and is just an example to build all other apis after. API files go to {html root}/api folder

General flow: JS file calls API PHP file, in its turn it calls backend PHP file which goes to DB with DMLs and queries.

Please write all API PHPs and all corresponding backend PHPs for "get surveys", "delete survey", "save survey" tasks. In doing so, please correctly match snake_case DB identifiers to camelCase JS/PHP identifiers.
